# Model Card

## Intended Use

Predict player performance metrics for fantasy sports analysis, sports analytics demos, and planning workflows across cricket, football, and NBA.

## Model Family

- Primary runnable model: scikit-learn ensemble with gradient boosting, random forest, and MLP components.
- Time-series/deep-learning support: `TorchLSTMRegressor` class is included for sequence-oriented experiments.
- Prediction intervals: quantile gradient boosting models estimate 80% lower and upper bounds.

## Training Data

The default dataset is deterministic synthetic sports data generated by `sports_predictor.data_collection.generate_sample_data`. It encodes relationships among player skill, usage, rest days, home/away context, opposition strength, injury risk, and seasonal form.

## Metrics

Run `python models.py` to produce `reports/validation_results.json` with:

- MAE
- RMSE
- MAPE
- R2
- MAE as percentage of average performance
- 80% interval coverage

## Limitations

Synthetic data proves the system pipeline works, but it does not prove real-world sports accuracy. Production use requires real historical datasets, leakage checks, sport-specific targets, monitoring, retraining, and calibration against live outcomes.

## Ethical And Safety Notes

Predictions are probabilistic and should not be presented as guarantees. Betting-related use should comply with local laws and platform policies.
